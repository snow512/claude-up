## 코드정리해

미푸시 커밋 또는 언스테이징 변경 파일을 대상으로 클린코드 작업을 수행합니다.
사용자가 "전체"라고 명시하면 프로젝트 전체를 대상으로 합니다.

---

### 1단계: 대상 파일 식별

```bash
# 미푸시 커밋의 변경 파일
git diff --name-only @{upstream}..HEAD 2>/dev/null || git diff --name-only origin/$(git rev-parse --abbrev-ref HEAD)..HEAD 2>/dev/null

# 언스테이징/스테이징 변경 파일
git diff --name-only
git diff --name-only --cached

# 언트래킹 파일
git ls-files --others --exclude-standard
```

- 위 결과를 합쳐 **중복 제거** → 대상 파일 목록
- `.env`, `package-lock.json`, `dist/`, `node_modules/`, 이미지/폰트 등 바이너리는 제외
- 사용자가 **"전체"**를 명시한 경우: `src/` 하위 전체를 대상으로 함

---

### 2단계: 전체 파일 읽기 + 분석

대상 파일을 **모두 읽고** 아래 7가지 관점으로 분석합니다.
Agent 도구를 활용하여 병렬로 분석하세요.

#### 체크 항목

| # | 카테고리 | 체크 내용 |
|---|----------|----------|
| 1 | **버그헌팅** | null/undefined 미처리, off-by-one, 비동기 에러 누락, race condition, 메모리 누수 (이벤트 리스너 미해제), 무한 루프 가능성 |
| 2 | **컨벤션** | 네이밍 일관성, 파일/함수 구조, import 순서, 불필요한 주석, 코딩 스타일 통일 |
| 3 | **예방적 수정** | 타입 안전성 부족 (`any` 남용), 경계값 미처리, optional chaining 누락, 빈 배열/객체 방어 |
| 4 | **로그 보강** | catch 블록에 에러 로그 누락, 중요 분기점 디버깅 정보 부재, 사용자 액션 추적 누락 |
| 5 | **중복 공통화** | 동일/유사 로직 반복, 매직 넘버/문자열, 공통 유틸로 추출 가능한 패턴 |
| 6 | **성능** | 불필요한 리렌더링, useMemo/useCallback 누락, N+1 쿼리 패턴, 대량 데이터 미페이지네이션 |
| 7 | **보안** | XSS 가능성, SQL 인젝션, 민감정보 노출, 인증/인가 누락 |

---

### 3단계: 발견사항 보고

분석 결과를 **카테고리별로 요약**하여 사용자에게 보고합니다.

#### 보고 형식

```
## 코드 정리 분석 결과

대상: {N}개 파일 (미푸시 커밋 {M}개 + 언스테이징 {K}개)

### 발견 사항 ({총 건수}건)

#### 🐛 버그헌팅 ({N}건)
- `파일:라인` — 설명

#### 📐 컨벤션 ({N}건)
- `파일:라인` — 설명

#### 🛡️ 예방적 수정 ({N}건)
- `파일:라인` — 설명

#### 📋 로그 보강 ({N}건)
- `파일:라인` — 설명

#### 🔄 중복 공통화 ({N}건)
- `파일:라인` — 설명

#### ⚡ 성능 ({N}건)
- `파일:라인` — 설명

#### 🔒 보안 ({N}건)
- `파일:라인` — 설명
```

- 발견 건수가 0인 카테고리는 생략
- 각 항목에 **심각도** 표시: `[critical]`, `[warning]`, `[info]`

---

### 4단계: 수정 수행

- 보고 후 **사용자 확인 없이** 바로 수정을 진행
- 단, `[critical]` 이슈가 있으면 먼저 사용자에게 알리고 수정 방향을 확인
- 수정 시 **기능 변경 없이** 코드 품질만 개선 (동작이 바뀌면 안 됨)
- 수정 완료 후 빌드 검증 (`vite build` 또는 해당 프로젝트 빌드 명령)
- 테스트가 있으면 테스트도 실행

---

### 5단계: 수정 요약

```
## 코드 정리 완료

수정: {N}건 / 스킵: {M}건
빌드: ✅ 성공
테스트: ✅ {통과}/{전체} 통과

### 수정 내역
- `파일` — 변경 내용 요약
```

---

### 주의사항

- **기능 변경 금지** — 동작이 달라지는 수정은 하지 않음
- **과도한 리팩토링 금지** — 파일 구조 변경, 대규모 추상화 도입 등은 하지 않음
- **커밋하지 않음** — 수정만 하고 커밋은 사용자가 별도로 요청
- 프론트엔드/백엔드 자동 감지하여 적절한 빌드/테스트 명령 사용
