## 관리 지침들 (아브라카다브라)

사용자가 직접 명령할 때 수행할 지침들. 나중에 슬래쉬 커맨드나 스킬로 만들어질 수 있음.

### 매직 커맨드 목록

- **"지침에 기록해"** = `CLAUDE.md`의 관리 지침들 섹션에 기록해.
- **"진행사항 파악해"** = `docs/worktrees/worktree-{현재워크트리}.md`와 커밋 기록 등을 읽고 `docs/tasks`에서 관련 타스크를 찾아서 읽고 현재 진행 상태를 파악한다.
- **"프로젝트 최신화해"** = git pull -p로 소스 및 브랜치를 최신화 하고, backend, frontend의 npm 모듈을 설치하여 프로젝트 구동 환경을 최신화 한다.
- **"타스크 기록해"** = 현재 작업중인 내용에 대한 `docs/tasks/`폴더의 관련 타스크를 업데이트 하거나 관련 타스크가 없으면 새로 생성해.
- **"타스크 완료해"** = 현재 작업중인 내용에 대한 `docs/tasks/`폴더의 관련 타스크를 업데이트 하고 완료처리해.
- **"진행상황 기록해"** = `docs/worktrees/worktree-{현재워크트리}.md`에 현재 워크트리의 브랜치·작업 내용만 기록.
- **"보강해"** = 기존의 구현 내용을 보강하거나 문제점을 찾아서 수정해. 그리고 동작이 잘되었는지 테스트까지 완료해. 필요시 관련 타스크나 진행사항을 먼저 업데이트하고 수정이 종료된 뒤에도 관련타스크나 진행사항을 업데이트해
- **"개선해"** = 기존의 구현 내용을 조금 확장하여 편리성을 증대시키거나 코드레벨이 아닌 사용성 레벨에서의 문제점까지 개선하라는 뜻. 반드시 개선할 내용을 먼저 사용자에게 보고 하고 지정된 개선 작업만 수행한다. 사용자가 선택한 내용만을 관련 타스크에 업데이트 하거나 타스크를 새로 작성한다.
- **"디베롭에 싱크해"** = 미커밋 변경이 있으면 커밋 → 푸시 → origin/develop을 현재 브랜치에 머지 → 현재 브랜치를 develop에 머지 → develop 푸시 → 현재 브랜치로 복귀. 양방향 머지로 커밋을 맞춘다.
- **"디벨롭에 머지해"** = `/merge-develop` 스킬을 실행해.
- **"커밋 푸쉬해"** 또는 **"커푸"** = `/commit-push` 스킬을 실행해.
- **"커밋 푸쉬만해"** = 현재 작업된 내용에 대해서 커밋하고 푸쉬만해. (타스크 업데이트 없이)
- **"서버재기동해"** = `/restart-server` 스킬을 실행해. 인자가 없으면 `backend/.env`의 PORT 값으로 환경(main/second/third)을 자동 감지.
- **"서버내려"** = `stop.sh`를 실행해. 환경은 `backend/.env`의 PORT로 자동 감지.
