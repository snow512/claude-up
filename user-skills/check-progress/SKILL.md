## 미커밋 확인해

미커밋 변경사항, 워크트리 진행 기록만을 이용하여 현재 진행 상태를 보고합니다.

## 진행상태 파악해
미커밋 변경사항, 워크트리 진행 기록, 관련 타스크 및 문서 등을 종합하여 현재 진행상태를 보고합니다.

### 실행 순서

1. **병렬 수집** — 다음을 동시에 실행:
   - `git status` (현재 브랜치명 + 변경 파일 목록)
   - `git diff --stat` + `git diff --stat --staged` (미커밋 변경 요약)
   - `git log --oneline -10` (최근 커밋 이력)

2. **워크트리 진행 파일 읽기**:
   - `docs/worktrees/worktree-main.md` 읽기
   - "진행 중인 작업" 섹션에서 현재 브랜치와 관련된 항목 파악

3. **관련 타스크 확인**:
   - `docs/tasks/` 폴더에서 현재 브랜치·작업과 관련된 task 파일을 Glob/Grep으로 검색
   - 관련 task 파일을 읽어 세부 체크리스트·Phase 진행 상태 파악
   - 완료된 항목(`.completed/`)은 제외

4. **미커밋 변경 상세 분석**:
   - 변경이 있으면 `git diff` 전문을 읽어 변경 내용 파악
   - 어떤 파일에서 어떤 작업이 진행되었는지 요약

5. **종합 보고** — 아래 형식으로 출력:

```
## 현재 진행 상황

**브랜치:** {브랜치명}
**워크트리:** main
**관련 타스크:** {타스크 파일명 또는 "없음"}

### 워크트리 진행 현황
- {worktree-(개별워크트리ID).md에서 현재 브랜치 관련 진행 요약}

### 관련 타스크 상태
- {타스크 파일의 Phase/체크리스트 진행 요약}

### 미커밋 변경사항
- {파일명}: {변경 요약}
- ...
(변경 없으면 "미커밋 변경 없음")

### 최근 커밋 이력
- {커밋 해시} {메시지}
- ...

### 다음 할 일
- {타스크·워크트리 기록 기반으로 남은 작업 또는 추천 다음 단계}
```

### 주의사항

- 현재 워크트리 md 파일만 참조 (다른 워크트리 파일 수정 금지)
- **보고만 수행하며 파일 수정은 하지 않음**
- 변경사항이 없으면 "미커밋 변경 없음"으로 표시
